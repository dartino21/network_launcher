# Network Launcher

Network Launcher запускает веб-проект на вашем компьютере и создаёт для него временный публичный HTTPS-адрес через ngrok. Программа подходит, чтобы показать сайт клиенту, протестировать его с телефона или дать коллегам ссылку на локальную разработку.

Она не загружает исходный код сайта на сервер: сайт продолжает работать на вашем компьютере. Network Launcher запускает локальный сервер, ставит перед ним proxy и направляет в proxy безопасный туннель ngrok.

## Быстрый старт

1. Скачайте `NetworkLauncher-windows-x64.zip` из раздела **Releases** этого репозитория.
2. Распакуйте архив, затем откройте `NetworkLauncher.exe`.
3. Один раз настройте [ngrok authtoken](https://dashboard.ngrok.com/get-started/your-authtoken). Самый простой способ для portable-версии — сохранить его в переменной Windows:

   ```powershell
   setx NGROK_AUTHTOKEN "ВАШ_ТОКЕН"
   ```

   После этого закройте и снова откройте программу. Если ngrok уже установлен в системе, можно вместо этого выполнить `ngrok config add-authtoken ВАШ_ТОКЕН`. Токен не сохраняется в конфигурации Network Launcher.
4. Нажмите **Выбрать папку**, укажите корень веб-проекта и нажмите **Запустить проект**.
5. Скопируйте появившийся публичный URL. Пока статус показывает «сайт запускается», не закрывайте программу.

Папка `data` рядом с EXE создаётся автоматически. В ней хранятся настройки, журнал `data/logs/app.log` и временные служебные файлы.

## Поддерживаемые сайты

| Маркер в корне папки | Что делает программа | Требование |
| --- | --- | --- |
| `index.html` | Запускает встроенный сервер статического сайта с SPA fallback. | Нет |
| `package.json` | Запускает `npm start`; если есть `build/`, использует `npx serve -s build`. | Node.js LTS и npm |
| `app.py` | Запускает Flask на локальном адресе. | Python и Flask; предпочтительно `.venv` внутри проекта |
| `docker-compose.yml`, `compose.yml` | Запускает сервисы через Docker Compose. | Запущенный Docker Desktop |

Выбирайте именно папку проекта, а не отдельный HTML-файл или EXE. Если в папке нет ни одного маркера, кнопка запуска останется недоступной.

## Как работает ссылка

1. Программа определяет тип сайта и проверяет зависимости.
2. Создаёт локальный proxy: он принимает единый входящий адрес.
3. Запускает ngrok и сразу показывает публичный HTTPS-URL.
4. Запускает выбранный сайт и ждёт его ответа на локальном порту.
5. Proxy передаёт обычные страницы во frontend, а backend-пути — в API. Поддерживаются HTTP, SSE и WebSocket.

Для сайта с frontend и API используйте относительные запросы, например `/api/products`. Жёстко прописанный в браузерном коде адрес `http://localhost:5000` у посетителей будет указывать на их собственный компьютер, поэтому такая функция через публичную ссылку не сработает. Программа показывает это как предупреждение в журнале.

## Настройки запуска

- **Порт** — предпочитаемый локальный порт frontend. Если он занят, выбирается свободный.
- **Автозапуск при выборе папки** — сразу запускает распознанный сайт после выбора папки.
- **Frontend** — `auto`, имя Docker-сервиса или порт frontend. Обычно оставьте `auto`.
- **Backend** — `auto`, `none`, имя Docker-сервиса или порт API.
- **Backend-пути** — список префиксов, которые proxy направляет в API: `/api`, `/health`, `/ws`, `/socket.io`, `/graphql`.
- **Dev/HMR** — режим совместимости для Vite, Next.js, Webpack и других dev-серверов с WebSocket/HMR. `Авто` — рекомендуемый вариант.
- **Host upstream** — определяет, передавать ли исходный публичный `Host` сервису. Оставьте `Авто`, если сайт не требует особой настройки.
- **Переменные public URL** — имена переменных окружения, которым во время запуска передаётся созданный URL, например `NEXTAUTH_URL` и `AUTH_URL`.

Настройки сохраняются отдельно для каждой выбранной папки сайта в `data/config.json`.

## Если запуск не работает

| Симптом | Что проверить |
| --- | --- |
| Публичной ссылки нет | Интернет, ngrok authtoken и журнал `data/logs/app.log`. |
| Ссылка появилась, но статус долго «сайт запускается» | Вкладку **Логи**: чаще всего сайт не слушает указанный порт, не установлен npm/Flask или Docker ещё собирает контейнер. |
| Node-сайт не стартует | Установите Node.js LTS, откройте папку проекта и проверьте `package.json`/скрипт `start`. |
| Flask-сайт не стартует из EXE | Создайте `.venv` в проекте и установите Flask: `python -m pip install flask`. |
| Docker-сайт не стартует | Откройте Docker Desktop и дождитесь статуса Running. |
| Страница открывается, но кнопки/API не работают | Замените browser-side `localhost` на относительные `/api/...` и проверьте Backend-пути. |

В программе те же подсказки доступны по кнопке **?**.

## Разработка и сборка

```powershell
python -m pip install -r requirements.txt
$env:PYTHONPATH = "src"
python -m network_launcher
python -m pytest
```

Для локального EXE: `powershell -ExecutionPolicy Bypass -File scripts\build_windows.ps1`.

Исходный код находится в `src/network_launcher`, тесты — в `tests`, документы — в `docs`. Подробный стартовый указатель: [START_HERE.md](START_HERE.md).
